# 血饮知识库

这是一个离线可运行的本地知识库原型，目标不是直接替代写作，而是先把历史文章沉淀成一个可检索、可引用、可继续扩展到 RAG 的底座。

当前版本做了三件事：

1. 扫描当前目录下的 `txt` 和 `docx` 文章。
2. 自动清洗文本、提取标题和日期、按段落分块。
3. 抽取文章级观点卡，并写入 SQLite 数据库和中文 FTS 索引。

## 目录

- `build_kb.py`: 构建知识库
- `query_kb.py`: 查询知识库并导出给大模型的上下文
- `prompt_templates/`: 后续写作时可直接复用的提示词模板
- `data/`: 构建出来的 SQLite 数据库

## 快速开始

在当前项目根目录执行：

```bash
python3 kb/build_kb.py --source .
```

构建完成后，查询示例：

```bash
python3 kb/query_kb.py "半岛 中美 金融" --limit 5
python3 kb/query_kb.py "美联储 加息 A股" --topic 金融货币 --limit 5
python3 kb/query_kb.py "伊朗 以色列 暗杀" --date-from 2023-01-01 --limit 6
```

导出一个可以直接喂给大模型的上下文文件：

```bash
python3 kb/query_kb.py "半岛 中美 金融" --limit 6 --export-context kb/data/half_island_context.md
```

## 适合你的实际工作流

建议把它当成三层结构：

1. 历史观点层
   用这个知识库检索血饮过去对某个主题的稳定框架、叙事模式、术语和推理路径。

2. 近期事实层
   另外补一层“最新事件资料”，例如新闻、政策、数据、讲话原文。

3. 生成层
   把“历史观点层 + 近期事实层”一起喂给模型，让模型输出新文章。

注意：历史文章只能提供“既有观点和语言风格”，不能替代“最新事实核验”。

## 数据表

SQLite 中有三张核心表：

- `articles`: 文章级元数据和全文
- `chunks`: 分块后的正文
- `chunks_fts`: 全文检索索引
- `article_views`: 观点抽取结果，包含核心结论、因果链、敌我判断、修辞特征

## 观点抽取

当前版本是规则抽取，适合先做知识沉淀，不依赖外部模型。

每篇文章会尽量提炼出：

- `core_claims`: 核心结论
- `causal_links`: 关键因果链
- `stances`: 敌我判断和角色定位
- `rhetoric_markers`: 常见修辞钩子

这层的价值在于：后续生成文章时，不必每次都把整篇原文喂给模型，可以先召回观点卡，再按需补原文证据。

## 下一步怎么扩展成更强的 RAG

当前版本优先保证离线、低依赖和可维护。后续可以继续加：

1. 向量检索
   追加 embedding 字段，把同义改写和隐含主题也找出来。

2. 观点抽取
   给每篇文章额外抽取“核心结论、敌我判断、关键因果链、常用修辞”。

3. 时间轴
   把“某一主题在不同年份的判断变化”整理成 timeline。

4. 双库结构
   一个库放“血饮历史文章”，另一个库放“近期新闻资料”，生成时分别召回。

## 风险提醒

这个知识库会帮助模型更像“血饮”地思考和写作，但不会自动保证事实正确、逻辑可靠或符合平台风险要求。涉及近期事件、人物、金融判断、公共卫生时，生成前必须单独补最新事实源。
