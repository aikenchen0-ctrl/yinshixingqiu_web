# 知识星球测试数据驱动执行提示词 05

## 任务目标

请用“测试数据驱动开发”的方式继续推进这个仓库中的知识星球模块，重点是后端管理和对应的用户视角联动。当前最大问题不是没有代码，而是很多功能没有被真实数据校验过，所以页面不像、接口不稳、逻辑容易偏。

## 仓库定位

- 管理端前端：`admin-web`
- 后端：`backend`
- 小程序端：`miniprogram`

## 对标对象

- 知识星球网页：`https://wx.zsxq.com/group/28882128518851`
- 管理后台：右上角“管理后台”

## 工作总原则

每做一个功能，都必须遵守下面闭环：

1. 先分析真实页面和接口
2. 先设计测试数据
3. 再实现代码
4. 再验证数据库、接口、前端展示
5. 最后记录偏差和下一步

## 测试数据设计要求

请为每个功能建立可重复验证的数据样本，至少覆盖：

- 标准正常数据
- 空数据
- 极限数量数据
- 状态冲突数据
- 权限差异数据

优先准备这些场景：

- 多个星球
- 多种成员身份
- 帖子有置顶、精选、删除、审核中
- 评论有多层互动表现
- 会员有生效中、待生效、已过期
- 管理员有不同权限等级

## 开发重点

1. 先从 `backend` 里检查数据库结构、Prisma 模型、接口返回
2. 再对照 `admin-web` 页面字段使用情况
3. 找出哪些页面因为测试数据不对导致“看起来不像”
4. 优先补齐这些页面所需的数据样本与接口逻辑
5. 再修页面结构与交互

## 验证动作

每完成一个功能，请明确执行这些校验：

- 数据库是否真的写入了预期测试数据
- API 返回是否符合页面需要
- 管理端列表页、详情页、编辑页是否展示正确
- 用户视角是否读取到了同一份数据

## 交付要求

请不要只汇报“做了什么”，还要给出：

1. 用了哪些测试数据
2. 每组数据验证了哪个功能
3. 验证结论是否通过
4. 哪些页面仍然不够像知识星球

## 约束

- 小程序相关代码必须遵守原生微信小程序约束
- 不要引入重型依赖
- 后端以快速落地、清晰可维护为优先
- 如果某些真实字段抓不到，要标明推断依据

请开始执行，并把测试数据作为每项工作的前置步骤。
