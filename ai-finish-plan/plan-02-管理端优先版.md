# 知识星球管理端优先执行提示词 02

## 任务背景

当前仓库已经有部分实现，但“后端管理”这一块复刻得不够像。你的首要任务是把知识星球的管理视角做得更接近真实系统，同时兼顾用户视角的字段和数据链路一致性。

## 代码位置

- 管理端前端：`admin-web`
- 后端：`backend`
- 小程序端：`miniprogram`

## 外部参考

- 真实知识星球网页：`https://wx.zsxq.com/group/28882128518851`
- 管理后台入口：网页右上角“管理后台”

## 你必须做的事

1. 优先分析管理后台真实页面的 DOM 结构、布局、筛选区、表格、详情抽屉、弹窗、表单
2. 优先分析真实 Network，请求路径、请求方法、分页参数、筛选字段、返回结构、状态字段
3. 回到 `admin-web` 和 `backend`，找出当前最不像的页面
4. 优先修复这个页面及其依赖接口，再逐步扩展到相邻页面
5. 每个功能都要提前设计测试数据，并在完成后核对数据库与页面显示

## 页面复刻要求

管理端优先贴近以下模式：

- 顶部筛选 + 列表表格
- 详情抽屉 / 详情页
- 新建 / 编辑表单
- 状态开关
- 批量操作
- 分页、搜索、标签筛选

## 数据一致性要求

不要只把 UI 画像。你必须同步确认：

- 接口字段是否足够支撑页面
- 数据库表结构是否能覆盖真实业务
- 列表、详情、编辑、状态切换是否形成闭环
- 用户视角和管理视角是否共享同一套真实数据模型

## 测试数据规则

实现每个功能前，先定义测试数据集，至少包括：

- 一个正常星球
- 一个空内容星球
- 一个有多管理员的星球
- 一个有审核中内容的星球
- 一个有付费会员和到期会员的样本

完成后必须验证：

- 数据库记录对不对
- 接口返回对不对
- 管理端页面展示对不对
- 用户侧看到的数据是否同步合理

## 输出方式

你最终需要交付：

1. `admin-web` 中的页面与组件修正
2. `backend` 中接口、数据模型、测试数据修正
3. 对照真实管理后台后的差异说明
4. 本轮完成项与下一轮优先项

## 执行原则

- 先修最不像的页面，再扩展
- 先保证能跑通，再继续做细节
- 先有真实数据闭环，再做视觉打磨
- 允许参考 Ant Design Pro / Vue Element Admin 的后台范式，但不能脱离知识星球真实结构

现在请直接开始，先做代码盘点和差异识别，再进入修复。
