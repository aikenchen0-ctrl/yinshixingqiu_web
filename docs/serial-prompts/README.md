# 串行提示词说明

这组提示词是为 `D:\CodeDevelopment\xueyinMiniapp` 准备的串行执行版。

不是 10 份内容重复的模板，而是 10 个前后衔接的步骤。推荐严格按顺序执行：

1. `01-项目盘点与范围冻结.md`
2. `02-后端现状审计与缺口清单.md`
3. `03-后端一期开发计划.md`
4. `04-后端一期落地执行.md`
5. `05-admin-web现状盘点与页面优先级.md`
6. `06-admin-web一期开发提示词.md`
7. `07-小程序现状盘点与联调前检查.md`
8. `08-小程序核心闭环补齐.md`
9. `09-三端联调与服务器核验.md`
10. `10-收尾验收与第二轮规划.md`

## 使用方式

- 一次只喂 1 份
- 上一份的结论作为下一份输入
- 不建议跳号执行
- 如果中途范围变化，优先回到 01 或 03 重新冻结范围和计划

## 这套提示词的设计逻辑

- 先盘点，再定范围
- 先后端，再后台，再小程序
- 最后联调、验收、规划第二轮

## 适用场景

- 你要逐轮推进这个项目
- 你要把任务拆给不同时间段执行
- 你想避免 AI 一次性铺太大、做着做着跑偏
