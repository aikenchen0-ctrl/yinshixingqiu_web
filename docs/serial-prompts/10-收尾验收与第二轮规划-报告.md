# 收尾验收与第二轮规划

## 第一轮目标回顾

第一轮冻结范围的核心不是把整个“知识星球”产品线一次性做完，而是先做出一条可以联调、可以继续迭代的主链路。结合 [01-项目盘点与范围冻结-报告](/mnt/hgfs/xueyinMiniapp/docs/serial-prompts/01-项目盘点与范围冻结-报告.md) 与当前代码状态，这一轮真正要解决的是：

- 让 `backend`、`admin-web`、`miniprogram` 围绕知识星球主链路形成同一套数据模型。
- 优先跑通登录、发现星球、加入星球、星球主页、发帖、评论、点赞、我的星球。
- 让管理端先具备最小可用能力，而不是完整复刻知识星球所有运营后台。
- 允许加入链路暂时保留 mock 支付，但要求订单、会员关系、页面联动至少形成工程闭环。

本轮额外形成的范围调整结论：

- 星球球主的运营后台按“最小后台”处理，不追求完整实现收入、续期、积分榜、活动工具、运营实验矩阵。
- 第二轮优先补稳定性、权限和最少量真实后台页面，不继续无边界扩张后台模块。

## 当前残留问题

### 1. 计划执行状态和代码实际状态脱节

`ai-finish-plan` 的批量执行记录仍停留在 `plan-01`，没有形成“1 到 10 全部跑完”的可追踪产物；但代码本身已经明显推进到了更后面的阶段。这意味着：

- 自动执行记录不能代表真实完成度。
- 当前必须以代码和基础检查结果为准，而不是以批跑器进度为准。

### 2. 管理端范围已经偏大

当前 `admin-web` 与 `backend` 中的后台能力已经覆盖收入、推广、成员、内容、续期、优惠券、权限、排行榜、活跃工具等多个方向。继续按这个方向平均推进，任务会明显失控。

这里需要明确范围收口：

- 第一优先级只保留“最小运营后台”。
- 其余后台模块作为第二轮或更后续的扩展项，不再视为当前必须完成。

### 3. 仍存在关键闭环缺口

- 支付闭环仍是 mock。
- 登录态、访问控制、后台权限收口仍应继续审查。
- 文档与代码之间存在明显时差，需要一份新的收尾报告统一口径。

### 4. 当前基础验证状态

本次检查的结论如下：

- `npm run check:backend` 通过。
- `npm run check:miniprogram` 通过。
- `npm run build:admin` 未通过。

当前 `admin-web` 构建失败不是业务代码语法错误，而是本地依赖环境缺少 `rolldown` 对应 Linux 原生 binding，需要重新安装依赖后再验证。

## 第二轮优先级建议

### P0：先收口范围

把星球球主后台收缩成最小范围，只保留真正影响联调闭环和日常运营判断的页面：

- 收入数据
- 推广数据 / 渠道二维码
- 成员管理
- 内容管理
- 权限设置

以下模块降为第二轮或可选项：

- 续期全套页面
- 积分榜
- 活跃工具
- 付费页优化
- Idea Lab
- 成员验证
- 频道直播 / 其他运营工具矩阵

### P1：继续补主链路稳定性

- 收紧 session 与用户身份来源。
- 继续补内容访问控制和管理权限校验。
- 处理管理端构建环境问题，恢复 `admin-web` 的可重复构建。

### P2：把最小后台做实

第二轮不追求后台面广，而是把下面 5 类页面做成真正可用：

- 收入数据
- 推广数据
- 成员管理
- 内容管理
- 权限设置

### P3：清理历史残留

- 清理旧版小程序页面和历史占位逻辑。
- 继续压缩“看起来像很多功能，实际上没有验收”的状态。
- 在代码完成后补正式验收文档，而不是再堆更多功能页。

## 第二轮建议拆分步骤

### 步骤 1：冻结最小运营后台范围

- 收缩菜单。
- 保留扩展页代码，但不再把它们作为当前主路径。
- 把第二轮工作重点放在 5 个核心后台页面上。

### 步骤 2：修稳定性和验证链

- 先解决管理端构建环境问题。
- 再补主链路中最影响联调的权限、身份、访问控制问题。

### 步骤 3：完成最小后台的真实联调

- 成员管理页
- 内容管理页
- 收入 / 推广页
- 权限设置页

要求是“能查、能看、能改、能解释”，而不是页面越多越好。

### 步骤 4：做真正收尾

- 形成新的执行结果文档。
- 把“已完成 / 未完成 / 风险 / 下一步”固定下来。
- 再决定是否进入续期、积分榜、工具页等扩展模块。

## 最终项目状态总结

当前项目不再是“从零做知识星球”，而是已经进入“主链路可跑、后台开始扩展、需要范围治理和收尾验收”的阶段。

最准确的阶段判断是：

- 第一轮主链路已基本形成。
- 第二轮扩展已经开始，但后台范围出现膨胀。
- 当前最需要做的不是继续铺更多后台页，而是收口范围、稳定主链路、把最小后台真正做实。

因此，接下来的执行原则应明确为：

- 运营后台只做最小可用版。
- 不再追求完整复刻全部球主后台。
- 每做一项，都优先服务主链路闭环和真实可验收结果。
